# GitHub 入门

## GitHub 是什么？

GitHub 是全球最大的代码托管平台，你可以把它理解为"程序员的社交网络"：

- **代码仓库** — 存放你的代码项目
- **开源社区** — 全球程序员在这里分享代码
- **版本控制** — 记录代码的每一次修改
- **协作工具** — 多人一起开发项目

## 为什么大学生要学 GitHub？

- **课程作业** — 很多编程课要求用 Git 提交作业
- **开源项目** — 参与开源项目是很好的学习方式
- **找工作** — 你的 GitHub 主页就是你的简历
- **备份代码** — 代码放在 GitHub 上不会丢
- **学习资源** — GitHub 上有海量开源项目和教程

## Git 和 GitHub 的区别

| Git | GitHub |
|-----|--------|
| 版本控制工具 | 代码托管平台 |
| 本地使用 | 在线服务 |
| 记录代码修改历史 | 存放和分享代码 |
| 命令行工具 | 网站 + 桌面客户端 |

简单说：**Git 是工具，GitHub 是平台**。你用 Git 来管理代码，用 GitHub 来存放代码。

## 什么是版本控制？

想象一下你在写论文：
- 你写了一版，保存为"论文_v1.docx"
- 改了一些内容，保存为"论文_v2.docx"
- 又改了一些，保存为"论文_v3_final.docx"
- 再改一些，保存为"论文_v3_final_真的最终版.docx"

这很混乱对吧？Git 就是来解决这个问题的。

### Git 怎么工作？

1. 你修改了代码
2. 你告诉 Git："保存这次修改"（commit）
3. Git 记录下：谁、什么时候、改了什么
4. 你可以随时回退到任何一次修改

```
修改 → 保存 → 修改 → 保存 → 修改 → 保存
  ↓      ↓      ↓      ↓      ↓      ↓
v1    v2     v3    v4     v5    v6
```

任何时候觉得不对，都可以回退到之前的版本。

## GitHub 能做什么？

### 1. 存放代码
把你的代码放在 GitHub 上，不怕电脑坏了丢代码。

### 2. 开源项目
全球程序员在这里分享自己的代码，你可以：
- **学习** — 看别人怎么写代码
- **使用** — 直接用别人写好的代码
- **贡献** — 帮别人改进代码

### 3. 协作开发
多人一起做一个项目：
- 每个人在自己的"分支"上开发
- 开发完合并到主分支
- Git 会自动处理大部分合并冲突

### 4. 展示自己
你的 GitHub 主页就是你的简历：
- 你参与过什么项目
- 你写过什么代码
- 你的代码质量如何

## GitHub 上有什么？

### 你可能会用到的

| 类型 | 举例 | 用途 |
|------|------|------|
| **开源项目** | Linux、Python、React | 学习、使用 |

| **工具** | VS Code、Chrome 插件 | 提高效率 |

- **developer-roadmap** — 开发者学习路线图
- **public-apis** — 免费 API 合集
- **awesome-python** — Python 优秀资源合集
